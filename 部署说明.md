## 🚀 部署到 GitHub Pages（免费）

### 第一步：创建 GitHub 仓库

1. 登录 https://github.com （没有的话先注册）
2. 新建仓库，仓库名必须是：**`你的GitHub用户名.github.io`**
   - 例如你叫 `zhangsan`，仓库名就是 `zhangsan.github.io`
   - 仓库设为 **Public**
3. 创建后，记录仓库地址（如 `https://github.com/zhangsan/zhangsan.github.io.git`）

### 第二步：推送代码

打开终端，进入博客目录，运行：

```bash
cd D:\test\security-blog

# 初始化 Git
git init
git add .
git commit -m "init blog"

# 推送到你的仓库（注意替换成你的用户名）
git remote add origin https://github.com/<你的用户名>/<你的用户名>.github.io.git
git branch -M main
git push -u origin main
```

### 第三步：开启 GitHub Pages

1. 进入你的 GitHub 仓库 → Settings → Pages
2. Source 选择 **GitHub Actions** 或 **Deploy from a branch (main)**
3. 等待 1-2 分钟，访问 `https://<你的用户名>.github.io`

## 📝 后续写新文章的步骤

```bash
# 1. 创建新文章
hexo new "文章标题"

# 2. 编辑 source/_posts/文章标题.md （写内容）

# 3. 本地预览
hexo server
# 浏览器打开 http://localhost:4000

# 4. 生成静态文件
hexo generate

# 5. 提交到 GitHub
git add .
git commit -m "new post: 文章标题"
git push
```

## ⚙️ 部署前别忘了改配置

打开 `_config.yml`，搜索并替换：

| 原内容 | 替换为 |
|--------|--------|
| `[你的名字]` | 你的真实姓名 |
| `<你的GitHub用户名>` | 你的 GitHub 用户名 |
| `<你的邮箱>` | 你的真实邮箱 |

打开 `_config.next.yml`，搜索 `GitHub:` 和 `E-Mail:`，填入真实信息。

打开 `source/about/index.md`，填入你的联系方式。

当你把仓库推到 GitHub 后，记得把 `_config.yml` 中的 `url` 从 `http://localhost:4000` 改为 `https://<你的用户名>.github.io`。
