文件上传漏洞：配置文件攻击

发表于 2025-07-01 更新于 2026-06-04 分类于 渗透测试

引言

文件上传漏洞是 Web 安全中最常见的高危漏洞之一。除了常规的图片马、后缀绕过之外，配置文件攻击是一类容易被忽视但威力极大的利用技巧。攻击者通过上传服务器配置文件（.htaccess、.user.ini、web.config 等），可以在不直接上传 Webshell 的情况下获取代码执行能力，绕过诸多防护措施。

本文将系统梳理基于配置文件的文件上传攻击手法，涵盖 Apache、IIS、Nginx 等主流服务器，并提供完整的防御方案。

一、.htaccess 配置文件攻击

1.1 背景

.htaccess 是 Apache 的分布式配置文件。当 AllowOverride 未设为 None 时，目录下的 .htaccess 可以被解析生效。攻击者若能上传此文件，即可修改该目录的解析规则。

1.2 AddType — 将任意后缀映射为 PHP

二、.user.ini 配置文件攻击

2.1 原理

PHP 5.3.0+ 在 CGI/FastCGI 模式下支持 .user.ini，用于覆盖 php.ini 中的部分配置。该特性不限于 Apache——Nginx、IIS 等同样受影响。

生效条件：user_ini.filename 为默认值 .user.ini 且 user_ini.cache_ttl > 0。

  • 关闭请求过滤：<fileExtensions allowUnlisted="true" /> 绕过上传限制

四、Nginx 配置文件攻击

4.1 机制差异

Nginx 没有目录级分布式配置文件——这是设计哲学上的差异，无法像 Apache 或 IIS 那样上传特定文件直接覆盖目录解析规则。但攻击面依然存在。

五、PHP-FPM 配置覆盖

5.1 动态 php.ini 注入

FastCGI 协议中的 PHP_VALUE 和 PHP_ADMIN_VALUE 参数允许动态覆盖 PHP 配置。若攻击者能控制 fastcgi_param（例如通过 SSRF 打 PHP-FPM 监听端口）：

5.2 PHP-FPM 端口暴露 + SSRF

部分运维将 PHP-FPM 监听 0.0.0.0:9000。若无防火墙限制，攻击者可通过 SSRF + Gopher 协议直接与 PHP-FPM 通信，发送构造的 FastCGI 包实现任意代码执行（常用 gopherus 生成 payload）。

六、竞态条件攻击

配置文件攻击中存在多个可利用的时间窗口：

  • .user.ini 的 user_ini.cache_ttl 默认 300 秒，上传后缓存期内配置不生效，攻击者可在此期间替换已上传的图片马内容，或通过高频并发请求 “撞” 上缓存刷新窗口
  • 上传接口 “先写临时文件再重命名” 的机制：若可猜测 /tmp/phpXXXXXX 路径，可在文件被删除前通过 LFI 包含
  • 多线程模型：一个线程持续上传 .user.ini 争取最快生效，另一个高频访问目标 PHP 等待触发

八、防御方案

8.1 文件上传控制

  • 白名单后缀：仅允许 .jpg、.png、.pdf 等业务所需后缀，拒绝一切配置相关后缀
  • 过滤关键文件名：.htaccess、.user.ini、web.config、*.conf 等（含大小写变体与 NTFS 流后缀）
  • 内容类型检测：对文件的真实 MIME 类型（magic bytes）进行校验，不信任客户端 Content-Type
  • 强制随机重命名：使用 UUID / 时间戳命名，避免保留原始文件名

8.3 纵深防御

  • 最小权限：Web 应用对上传目录仅授予写入权限，不给执行权限
  • 目录隔离：将上传文件存储在 Web 根目录之外，通过独立脚本读取输出
  • 对象存储分离：使用 OSS / S3 等独立服务，避免文件直接落在应用服务器磁盘
  • WAF 规则：配置检测配置文件上传的规则（文件名关键词 + 内容特征）
  • 定期审计：扫描上传目录是否存在 .htaccess、.user.ini 等异常文件

8.4 监控告警

  • 记录所有文件上传操作，特别是以 . 开头的隐藏文件
  • 监控上传目录中非预期文件后缀的出现
  • 对短时间内高频访问同一 PHP 文件的行为告警
  • 审计 PHP-FPM 错误日志中异常的 auto_prepend_file 变更

九、常见误区

误区 真相
Content-Type 头不可信 由客户端发送，可任意伪造
黑名单后缀过滤足够 大小写、双后缀、NTFS 流等多手段可绕过
上传目录无 .php 所以安全 .user.ini 使图片马在访问其他 PHP 时执行
Nginx 没有配置攻击面 仍受 .user.ini 影响，且 include 配置不当可被利用
CDN / WAF 能完全防御 高级混淆和分段上传可绕过基于签名的检测
只禁 .htaccess 就够了 .htpasswd、.htgroup、.user.ini 等其他隐藏文件同样危险

结语

配置文件攻击是文件上传漏洞利用链中的高级技巧，往往不被常规安全测试覆盖，因此成为实战渗透中的关键手段。理解各类 Web 服务器的配置机制不仅有助于发现漏洞，更能指导团队从架构层面做好防御。

安全从业者应在授权的渗透测试中充分验证此类攻击面，推动配置文件防护纳入安全基线检查项，构建纵深防御体系。

后渗透：数据窃取与外传

发表于 2025-07-01 更新于 2026-06-04 分类于 渗透测试

前言

在后渗透阶段，攻击者已获得目标网络立足点，核心任务之一便是数据窃取与外传（Data Exfiltration）。无论窃取的是数据库凭证、商业机密还是PII，攻击者都需要将数据从内网安全传输到外部，同时规避DLP、IDS/IPS和流量审计。

本文梳理主流数据外传技术——分割压缩加密传输、DNS隧道、HTTP分块、ICMP隧道、云存储中转和反检测打包，并从防守视角介绍Wireshark流量分析检测方法。

二、分割压缩加密传输

直接传输大文件极易触发DLP，攻击者采用”先压缩、再分割、后加密”策略：

将加密分卷通过不同通道交错发送，降低单通道流量异常风险：分卷1走HTTPS POST（主通道），分卷2走DNS隧道（备用通道），分卷3走WebSocket（辅助通道）。

三、DNS隧道数据外传

DNS查询通常不被防火墙拦截。数据编码进子域名，由攻击者控制的权威DNS服务器接收。

四、HTTP POST分块传输

分块传输编码（Chunked Transfer Encoding）将数据拆分发送，模拟正常Web交互。

带抖动与加密的分块外传

流量伪装： User-Agent模仿主流浏览器、构造自然Referer链、Cookie模拟已认证会话；MIME类型变形（image/png、text/css）；请求间隔0.5-5秒随机抖动。

五、ICMP隧道数据窃取

ICMP报文不被应用层防火墙深度检查。攻击者利用Echo Request/Reply的Data字段承载数据（最多1472字节/包）。

发送端与接收端

绕过要点：控制发包速率（>20包/秒触发告警）；Payload填充”abcdef…”伪装Windows默认ping；使用ICMP type 0替代type 8逃避出站规则。

六、云存储桶作为中转站

利用合法云存储服务（S3、Azure Blob、阿里云OSS）作为中转，流量特征与正常业务一致。

DNS隧道检测

特征： FQDN > 52字符；标签高熵（Base64特征）；短时间内大量不同子域名。

HTTP POST外传检测

特征： 固定间隔周期性POST；请求体熵值高；Content-Type为octet-stream无业务对应。

ICMP隧道检测

特征： Payload异常大（正常32-56B，外传达1400+）；负载无固定模式；短时间大量不同Payload。

TLS侧信道检测

指标： 连接时长/数据量不匹配；SNI与IP地理位置不匹配；自签名证书；上传量远超下载量。

九、防御建议

网络层： DNS白名单、ICMP Payload限56B、DNS安全方案（Umbrella）、NetFlow基线、TLS Inspection。
主机层： EDR网络行为监控、限制Raw Socket、文件完整性监控、进程创建审计（Event ID 4688）。

后渗透：横向移动技术深度解析

发表于 2025-06-01 更新于 2026-06-04 分类于 渗透测试

引言

横向移动（Lateral Movement）是后渗透阶段的核心环节。攻击者在获得内网初始立足点后，需要穿越网络边界、扩大控制范围，最终抵达高价值目标。本文系统梳理主流横向移动技术的原理、工具链、代码示例及检测策略，为红蓝双方提供实战参考。

免责声明： 本文所述技术仅供安全研究与授权测试使用。未经授权对计算机系统实施渗透测试属于违法行为，违者将承担法律责任。作者不对任何滥用行为负责。

一、SMB 协议族横向移动

SMB（Server Message Block）是 Windows 内网最常见的横向通道，基于 445 端口进行认证和数据传输。

1.1 PsExec

PsExec 是 Sysinternals 工具，通过 SMB 上传 psexesvc.exe 到目标 ADMIN$ 共享，注册并启动 Windows 服务执行命令。

原理： ① 连接目标 IPC$ 完成认证；② 将 psexesvc.exe 写入 \\target\ADMIN$\system32\；③ 通过 SCM 远程创建服务；④ 服务创建命名管道转发 I/O。

检测特征： 服务名 PSEXESVC 是强特征，服务创建事件 ID 7045，ADMIN$ 下写入可执行文件。

1.2 SMBExec

SMBExec 是 Impacket 套件中的纯 Python 实现，相比 PsExec 更灵活——服务名随机化，采用“写入-执行-读取-删除”一次性模式。

原理： ① SMB 认证目标主机；② 往 ADMIN$ 写入批处理文件（%TEMP%\execute.bat）；③ 创建随机名服务执行 bat；④ 输出重定向到临时文件，读取后清理。

维度 PsExec SMBExec
实现方式 上传 exe + 持久管道服务 写入 bat + 一次性服务
隐蔽性 低（固定服务名 PSEXESVC） 中（随机服务名，自动清理）
交互式 Shell 支持 支持
日志痕迹 7045 + 文件写入 7045 + bat 执行后删除
检测难度 低 中

二、WMI 横向移动

WMI 通过 DCOM（135 + 动态高端口）或 WinRM（5985/5986）通信，是 Windows 环境中最常用的远程管理协议之一。

2.1 WMIExec

利用 WMI 的 Win32_Process 类远程创建进程执行命令。

原理： ① DCOM 认证目标（135 端口协商）；② 调用 Win32_Process.Create 远程进程创建；③ 目标 wmiprvse.exe 承载命令执行；④ 输出通过 SMB 共享回传。

检测特征： wmiprvse.exe 下派生子进程；WMI 活动 EventId 5861；进程创建审计 4688。

三、计划任务横向移动

3.1 ATExec

Windows XP/Server 2003 时代的 at 命令，通过 DCE/RPC 调用 atsvc 接口创建远程计划任务。

检测特征： 使用已弃用的 atsvc 命名管道；EventId 4698；任务名固定格式 At[1-99]。

3.2 schtasks

现代 Windows 的 schtasks 支持更灵活的任务配置，是 ATExec 的替代方案。

隐蔽技巧： 任务名伪装为 WindowsUpdate 等常见名称，任务描述仿写微软风格文案。

维度 ATExec schtasks
协议接口 DCE/RPC atsvc DCE/RPC ITaskScheduler
权限 强制 SYSTEM 可指定
触发器灵活度 低（仅固定时间） 高（多种触发条件）
适用系统 XP/2003 Vista+
日志 4698 + atsvc 管道 4698/4702 + taskschd 管道
检测难度 中（弃用特性明显） 中（需结合参数分析）

四、WinRM 与 PowerShell PSSession

WinRM 是 WS-Management 协议的微软实现，默认端口 5985（HTTP）和 5986（HTTPS），是远程管理的官方推荐方案。

WinRM 优势在于流量加密（HTTPS）且日志规范，但常被蓝队忽视其横向利用风险。

检测特征： EventId 4104（脚本块日志）、53504（WinRM 认证）、8001（PSSession 创建）；进程 wsmprovhost.exe。

五、RDP 劫持与 tscon 会话接管

Windows 远程桌面支持多会话（非 Server 版仅一个活动会话），在 SYSTEM 权限下可利用 tscon 劫持已有会话。

检测特征： tscon 进程执行；非典型用户登录已有会话；EventId 4778/4779 异常序列。

六、DCOM 横向移动（MMC20.Application）

DCOM 通过 135 端口协商后动态分配高端口，允许跨机器 COM 对象调用，监控覆盖度相对较低。

原理： ① 远程激活 MMC20.Application 对象（CLSID: 49B2791A-B1AE-4C90-9B8E-E860BA07F889）；② 执行 ExecuteShellCommand 方法；③ 命令由 mmc.exe 进程承载。

十、防御与缓解措施

  1. 权限治理： 最小权限原则，限制域管理员登录范围；部署 Microsoft LAPS 管理本地管理员密码阻断哈希传递；敏感主机加入 Protected Users 组。
  2. 网络分段： 按业务需求划分 VLAN，内网主机间限制 SMB/135/445/WinRM/RDP 协议；RDP 强制 Network Level Authentication（NLA）。
  3. 日志与监控： 全量收集 Sysmon（EventId 1、3、11、13）+ PowerShell 脚本块日志；对 ADMIN$/IPC$ 写入设置告警；监控 wmiprvse.exe、mmc.exe、tscon.exe 网络活动。
  4. 端点防护： 启用 Credential Guard 防止凭证转储；配置 Windows Defender Firewall 出站管控；AppLocker/WDAC 限制可执行文件路径。
